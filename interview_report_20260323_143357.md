## 二、参考答案（本场涉及要点）

### Cross-Encoder vs Bi-Encoder（方向 3 未答出）

<a id="a-cross-encoder"></a>

| | Bi-Encoder | Cross-Encoder |
|--|-----------|--------------|
| 编码方式 | Query 与 Document **分别**编码为向量，再算相似度 | Query 与 Document **拼接**后 **联合编码**，充分交互 |
| 规模 | 文档向量可 **离线预计算**，查询时近似 O(1) 检索 | 每个 (Query, Chunk) 对都要 **实时前向**，O(候选数) |
| 典型用途 | **粗召回**（全库/大规模） | **精排**（Top-N 小候选集） |

**为何精排常用 Cross-Encoder 作强基线**：交互式建模通常比双塔点积更准；但候选一大成本就爆，故不能替代向量粗排。

### 可插拔 Embedding（与仓库对齐）

<a id="a-可插拔架构"></a>

1. **新建** `src/libs/embedding/<your_provider>.py`：继承 `BaseEmbedding`，实现约定方法（如 `embed_texts`）。  
2. **修改** `src/libs/embedding/embedding_factory.py`：按现有写法为 `provider` 增加分支或映射（模板中写 `factory.py` 以你仓库文件名为准）。  
3. **修改** `config/settings.yaml` 中 `embedding` 段（如 `provider`、密钥、模型名）。
